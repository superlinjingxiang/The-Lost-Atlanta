# 失落的亚特兰大 Demo 开发进度 - 2026-06-25

本文档是今天开发上下文的压缩版。后续继续开发时，优先读取 `AGENTS.md`、`开发规范.md` 和本文档。

## 今日目标

- 把项目从一次性 Demo 脚本，逐步调整为可扩展的“全局阶段 + Hook + 系统解耦”方向。
- 完成可进入游戏、可返回主菜单、可继续游戏的基础闭环。
- 开始打磨主菜单、游戏内 UI、昼夜切换表现和美术资源接入。

## 已完成内容

### 架构与规范

- 新增并确认项目开发规范：后续代码必须保持可扩展、低耦合，禁止把天气、背包、商店、事件等未来系统写成一次性脚本。
- 当前默认架构方向：
  - `PhaseManager` 管理全局阶段与 Hook。
  - `GameManager` 管理核心游戏状态与流程入口。
  - `SaveManager` 管理存档读写。
  - UI 只负责展示和输入，不承载核心业务逻辑。
- 已开始落地 `PhaseManager`，后续天气、背包、商店、事件、NPC 等系统应优先通过阶段 Hook 接入。

### 游戏流程

- 主菜单支持：
  - 开始新游戏。
  - 检测存档后继续游戏。
  - 退出确认。
- 游戏内支持：
  - 显示天数、阶段、食物、精神、体力、行动格。
  - 白天选择行动：搜索物资、休息整备、探索街区、加固据点。
  - 行动消耗行动格，夜晚事件后推进到下一天。
  - 返回主菜单时保存当前进度。
  - 回到主菜单后可以点击继续游戏返回当前存档。

### UI 与交互修复

- 修复游戏内文字没有居中的问题。
- 游戏内左上角使用城堡图标作为“返回主菜单”按钮。
- 返回按钮位于主 UI 上层，当前可点击并会保存进度后返回主菜单。
- 游戏主界面采用居中内容宽度，避免在宽屏下内容被拉得过散。
- UI 修改时需要继续注意多分辨率兼容，避免写死全屏绝对布局。

### 昼夜切换表现

- 已接入太阳、月亮 PNG 素材。
- 白天/黑夜切换时，太阳或月亮沿屏幕上方弧线滑动。
- 弧线轨迹要求：
  - 从屏幕左侧进入。
  - 经过上方中央最高点。
  - 从屏幕右侧离开。
  - 中央速度较慢，边缘速度较快。
- 已取消太阳/月亮自身旋转。
- 素材已处理为透明背景，避免显示成方形底。

### 主菜单美术

- 主菜单背景图只用于主菜单，进入游戏后不使用。
- 新增主菜单城堡图标，已处理为透明背景并放入主菜单标题上方。
- 游戏内返回主菜单按钮使用白色城堡图标。
